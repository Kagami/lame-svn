The Winamp Decoding Bug


100only.wav            A  pure 100 Hz sine wave
100only.mp3            Encoded with "lame -h"

sweep.wav              A sine wav sweep from 20Hz-20,000Hz
sweep.mp3              Encoded with "lame -h"


The LAME project receives reports of this bug, in various forms, about once a week.  The problem seems to be that the LAME produced mp3 files of simple harmonics like the pure 100Hz tone shown below is decoded incorrectly with Winamp/Nitrane.  Testing shows that this is in fact not a bug in the LAME produced mp3 file, but a bug in the Nitrane decoding engine used by Winamp.  Most other decoders can play back the mp3 files without any problems. 

More evidence of bugs in Nitrane can be found in Matthew Loyd's post in the mp3encoder mailing list.  

Naoki Shibata has also documented some bugs in Nitrane, and has produced a Winamp decoder plugin based on mpg123.  See this web site.
