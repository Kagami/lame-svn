The Winamp  Decoding Bug

This bug is in Winamp 2.62 (latest version as of 5/1/00) and earlier versions.

Update 6/9/00:  Winamp 2.64 has the same bugs.
 

The LAME project receives reports of this bug, in various forms, about once a week.  The problem seems to be that the LAME produced mp3 files of simple harmonics like the pure 100Hz tone shown below is decoded incorrectly with Winamp/Nitrane.  Testing shows that this is in fact not a bug in the LAME produced mp3 file, but a bug in the Nitrane decoding engine used by Winamp.

The followin players will decode the MP3 file with no problems:  mpg123, xmms, freeamp (Xing engine),  musicmatch  jukebox, windows media player, sonique 1.30.4, UltraPlayer 1.0.

More evidence of bugs in Nitrane can be found in Matthew Loyd's post in the mp3encoder mailing list.

Naoki Shibata has also documented some bugs in Nitrane, and has produced a Winamp decoder plugin based on mpg123.  This plugin replaces Nitrane.   See Naoki's  web site.

Even more documentation of winamp bugs can be found at this MP3 Player Quality Comparson Site.   They have doecumented severe flaws in several popular decoders.
