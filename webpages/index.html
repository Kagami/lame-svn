The LAME Project

official home page  and sourceforge mirror.

In the great history of GNU naming, LAME stands for LAME Ain't an Mp3 Encoder.   LAME is not an mp3 encoder.  It is a GPL'd patch against the dist10 ISO demonstration source.  LAME is totally incapable of producing an mp3 stream. It is incapable of even being compiled by itself. You need the ISO source for this software to work.  The ISO demonstration source is also freely available, but any commercial use (including distributing free encoders) may require a patent license.

LAME is an educational tool to be used for learning about MP3 encoding.  The goal of the LAME project is to improve the psycho acoustics, noise shaping and speed of MP3.  Another goal of the LAME project is to use these improvements for the basis of a  patent free audio compression codec for the GNU project.

In Beta:

  • MPEG 2.5 support
  • No more patching!  Full souce code distribution since all ISO has been replaced!
  • As always, more speed and better quality.

LAME v3.70 - April 6 2000:

  • Many improvements in quality in speed over LAME 3.51!  see history.
  • MPEG1 layer III and MPEG2 layer III.   (MPEG2.5 in beta version)
  • GPSYCHO: a GPL'd psycho acoustic and noise shaping model.  Variable bitrate (VBR), joint stereo,  great pre-echo control and much improved bit allocation.
  • Quality is substantially better than ISO psy-model based encoders and close to that of the FhG encoder.
  • Fast!   Encodes faster than real time on a PII 266 at highest quality mode.
  • MP3x:  a GTK/X-Window MP3 frame analyzer for both .mp3 and unencoded audio files.

LAME 3.x uses software from:

  • Decoding engine (for the frame analyzer):  mpglib, from the mpg123 project.  Mpg123 is an excellent  MPEG player.  Mpglib is released under the GPL.
  • Input from almost any type of sound file now handled by the GPL'd libsndfile.
